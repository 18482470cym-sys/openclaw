# skill: daily-report — 每日工作汇报生成器

## 描述

根据今日对话记录生成结构化工作日报。

## 触发场景

用户表达以下意图时使用本技能：

- 生成日报、今日总结、今日工作汇报、写一下今天的日报、总结今天做了什么

## 执行逻辑

1. **获取今日对话**

- **当前会话**：当前对话中的今日消息已在上下文中，直接基于本轮对话提取工作相关内容。
- **可选（跨会话）**：若需汇总多个会话的今日内容，使用 `sessions_list`（如 `activeMinutes: 1440` 表示 24 小时内活跃）列出会话，再用 `sessions_history` 对目标会话（如 main）拉取近期消息；需配置允许 `sessions_list`、`sessions_history`。

2. **提取工作相关内容**

- 从消息中识别：任务/需求、进展、结论、待办、会议/沟通对象、关键决策、阻塞与风险。
- 忽略与工作无关的闲聊、问候、重复内容。

3. **生成结构化日报**

- 输出为 Markdown，建议结构：
  - **今日完成**：已完成事项（可带简要结果）。
  - **进行中**：未完成但已有进展的事项。
  - **待办/计划**：明日或后续要做的事。
  - **问题/风险**（可选）：阻塞、依赖、需协调项。
- 语言与用户一致（中文用户用中文，英文用英文）；若未明确则默认中文。

## 注意事项

- 不编造未在对话中出现的内容；无法从上下文中推断的项可标为「未提及」或省略。
- 若当前上下文中几乎没有今日对话，可提示用户「当前没有足够的今日对话可总结」，并建议在对应会话中再说一次「生成日报」或先补充对话后再生成。
